# Contributing to GP_ANA

We welcome contributions to the GP_ANA project! Here's how you can help:

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** to your local machine
3. **Create a new branch** for your feature or bug fix
4. **Make your changes** and test them
5. **Submit a pull request** with a clear description of your changes

## Contribution Guidelines

### Code Style
- Follow Python's PEP 8 style guide
- Write clear, concise code with comments where necessary
- Use meaningful variable and function names

### Pull Requests
- Keep pull requests focused on a single feature or bug fix
- Provide a clear description of what your changes do
- Include any relevant issue numbers in your pull request description
- Make sure all tests pass before submitting

### Reporting Issues
- Use the GitHub issue tracker to report bugs or suggest new features
- Include a clear description of the issue
- Provide steps to reproduce the problem if reporting a bug
- Include any relevant error messages or screenshots

### Feature Requests
- Clearly describe the feature you'd like to see
- Explain why this feature would be useful
- If possible, provide examples of how the feature would work

## Development Workflow

1. **Setup development environment**:
   ```bash
   git clone https://github.com/your-username/GP_ANA.git
   cd GP_ANA
   pip install -r requirements.txt
   ```

2. **Create a new branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes** and test them thoroughly

4. **Commit your changes**:
   ```bash
   git add .
   git commit -m "Add feature: your feature description"
   ```

5. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a pull request** on GitHub

Thank you for contributing to GP_ANA!
