# data_collector.py 程序说明

## 1. 数据源

该程序从以下数据源获取股票数据：

| 数据源 | 描述 | 使用的函数/API |
|-------|------|--------------|
| 新浪财经 | 主要数据来源，提供A股实时行情和历史行情数据 | `https://finance.sina.com.cn/`、`https://vip.stock.finance.sina.com.cn/`、`https://quotes.sina.cn/` |
| akshare | 提供数据获取的工具库，包含各种URL和常量定义 | `zh_sina_a_stock_url`、`zh_sina_a_stock_hist_url`、`zh_sina_a_stock_hfq_url`、`zh_sina_a_stock_qfq_url`、`zh_sina_a_stock_amount_url` |

## 2. 抓取的数据

程序为每只股票抓取以下数据：

### 2.1 实时行情数据
- 所有A股的实时行情数据，包括代码、名称、最新价、涨跌额、涨跌幅等

### 2.2 历史行情数据
- 个股的历史行情数据，支持复权处理（前复权、后复权）
- 包含日期、开盘价、最高价、最低价、收盘价、成交量、成交额等

### 2.3 CDR个股历史行情数据
- CDR个股的历史行情数据
- 包含日期、开盘价、最高价、最低价、收盘价等

### 2.4 分钟级行情数据
- 股票及股票指数的分钟级行情数据
- 支持1、5、15、30、60分钟的数据

## 3. 保存的文件

程序将数据保存到以下文件中：

| 文件名 | 保存路径 | 数据内容 |
|-------|---------|---------|
| `{ticker}_qfq.csv` | `{DATA_DIR}/{ticker}/` | 前复权的历史行情数据，包括日期、开盘价、最高价、最低价、收盘价、成交量、成交额等 |

其中，`{ticker}` 是股票代码（如 600519.SH），`{DATA_DIR}` 是从 config.py 导入的数据目录路径。

## 4. 数据处理特点

1. **增量更新**：如果本地已有数据，只获取新数据并追加，避免重复数据。
2. **错误处理**：程序对每个数据获取步骤都进行异常捕获，确保某个数据获取失败不会影响整个程序运行。
3. **随机延迟**：在每次数据请求之间添加随机延迟（2-4秒），避免请求过于频繁被 API 限制。
4. **复权处理**：支持前复权和后复权数据的获取和处理。
5. **数据排序**：按日期排序，确保数据的时间顺序正确。
6. **数据去重**：自动去除重复数据，确保数据的一致性。
7. **日期处理**：对日期数据进行解析和处理，确保数据的一致性。

## 5. 使用方法

1. **处理指定股票**：
   ```bash
   python data_collector.py --ticker 600519.SH
   ```

2. **处理配置文件中的所有股票**：
   ```bash
   python data_collector.py
   ```
   其中，股票列表从 config.py 中的 STOCK_TICKERS 获取。

## 6. 代码结构

- **_get_zh_a_page_count 函数**：获取所有股票的总页数。
- **stock_zh_a_spot 函数**：获取所有A股的实时行情数据。
- **stock_zh_a_daily 函数**：获取个股的历史行情数据，支持复权处理。
- **stock_zh_a_cdr_daily 函数**：获取CDR个股的历史行情数据。
- **stock_zh_a_minute 函数**：获取股票及股票指数的分钟级行情数据。
- **main 函数**：主函数，处理命令行参数并调用相应的数据获取函数。

该程序设计合理，结构清晰，能够有效地从新浪财经获取股票数据，并将数据保存到相应的文件中，为后续的分析和处理提供了便利。
