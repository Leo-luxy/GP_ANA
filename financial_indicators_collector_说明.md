# financial_indicators_collector.py 程序说明

## 1. 数据源

该程序从以下数据源获取股票财务指标数据：

| 数据源 | 描述 | 使用的函数/API |
|-------|------|--------------|
| akshare | 主要数据来源，提供多种财务相关数据 | `ak.stock_financial_report_sina`、`ak.stock_financial_analysis_indicator`、`ak.stock_financial_abstract` |

## 2. 抓取的数据

程序为每只股票抓取以下数据：

### 2.1 现金流量表数据
- 从新浪财经获取股票的现金流量表数据
- 只保留最近12组数据

### 2.2 利润表数据及指标
- 从新浪财经获取股票的利润表数据
- 计算毛利率（(营业收入 - 营业成本) / 营业收入 * 100）
- 计算净利率（净利润 / 营业收入 * 100）
- 只保留最近12组数据

### 2.3 资产负债表数据及指标
- 从新浪财经获取股票的资产负债表数据
- 计算资产负债率（负债合计 / 资产总计 * 100）
- 计算流动比率（流动资产合计 / 流动负债合计）
- 只保留最近12组数据

### 2.4 成长指标
- 基于利润表数据计算
- 营收增长率（同比）
- 净利润增长率（同比）
- 自动处理时间维度一致性问题

### 2.5 财务分析指标
- 从akshare获取财务分析指标
- 如果获取失败，尝试获取财务摘要数据

## 3. 保存的文件

程序将数据保存到以下文件中：

| 文件名 | 保存路径 | 数据内容 |
|-------|---------|---------|
| `{ticker}_financial_indicators.json` | `{DATA_DIR}/{ticker}/` | 包含所有财务指标数据，包括现金流量表、利润表、资产负债表、成长指标和财务分析指标 |

其中，`{ticker}` 是股票代码（如 300433.SZ），`{DATA_DIR}` 是从 config.py 导入的数据目录路径。

## 4. 数据处理特点

1. **错误处理**：程序对每个数据获取步骤都进行异常捕获，确保某个数据获取失败不会影响整个程序运行。
2. **随机延迟**：在每次数据请求之间添加随机延迟（2-4秒），避免请求过于频繁被 API 限制。
3. **数据筛选**：只保留最近12组财务数据，减少数据量。
4. **智能计算**：自动计算各种财务指标，并处理时间维度一致性问题。
5. **多数据源尝试**：获取财务分析指标失败时，尝试获取财务摘要数据。
6. **日期处理**：使用自定义的 DateEncoder 类处理日期对象，确保日期数据正确序列化。

## 5. 使用方法

1. **处理指定股票**：
   ```bash
   python financial_indicators_collector.py --ticker 300433.SZ
   ```

2. **处理配置文件中的所有股票**：
   ```bash
   python financial_indicators_collector.py
   ```
   其中，股票列表从 config.py 中的 STOCK_TICKERS 获取。

## 6. 代码结构

- **DateEncoder 类**：自定义JSON编码器，处理日期对象。
- **get_stock_financial_indicators 函数**：获取单个股票的财务指标，包括现金流量表、利润表、资产负债表、成长指标和财务分析指标。
- **main 函数**：主函数，处理命令行参数并调用 get_stock_financial_indicators 函数。

该程序设计合理，结构清晰，能够有效地从 akshare 获取股票财务指标数据，并将数据保存到相应的文件中，为后续的分析和处理提供了便利。
